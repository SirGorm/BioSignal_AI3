## Executive summary

The pipeline is methodologically clean on the parts that matter most for sanity: streaming code is genuinely causal (Welford, persisted `zi`, no `filtfilt`/`savgol`/whole-signal `find_peaks`), CV groups on subject name (not recording_id) for the NN side via `configs/splits_loso.csv`, and ECG/EDA exclusion is enforced by both the features dataset (`EXCLUDED_FEATURE_PREFIXES = ('ecg_', 'eda_')`) and the raw dataset (`RAW_CHANNELS = ['emg', 'ppg_green', 'acc_mag', 'temp']`). The big problems are (1) **fairness of the LightGBM/RF baseline ↔ NN comparison is broken** — the RF baseline uses a different splits file (`configs/splits_per_fold.csv`) that excludes recording_003 (Vivian) entirely and groups on `recording_id` rather than `subject_id`, while the NN pipeline uses `configs/splits_loso.csv` over all 10 recordings — so the headline "RF beats NN" comparison in `runs/FINAL_REPORT.md` is on different data; (2) **CLAUDE.md has drifted from code** in ~10 places (claims 9 recordings, the project actually has 10; baseline window claimed 90–120 s but `window_features.py` uses 60 s; `RandomForest` vs `LightGBM` naming is mixed); and (3) **fatigue results are stuck near the dummy floor** (per-set RF MAE 0.99 vs 1.013 dummy, per-subject median Pearson 0.46) — close to the literature ceiling implied by RPE↔lactate r ≈ 0.6–0.8 (Borg 1982, Robertson et al. 2003), but the per-subject signal is bimodal (Tias r ≈ +0.84 / Hytten r ≈ −0.62) which suggests RPE labelling reliability rather than feature limits is the bound. The dataset itself is small but consistent: 10 recordings × 10 subjects, ~2.15 M @ 100 Hz aligned-feature rows, 114 per-set RPE rows.

## Phase 0 — Discovery

### Factual pipeline description (from code)

The training-time pipeline is built around three on-disk stages plus a streaming counterpart:

1. **Dataset assembly** — [`scripts/build_dataset_aligned.py`](scripts/build_dataset_aligned.py) builds `dataset_aligned/recording_NNN/` by copying biosignal CSVs verbatim from `dataset/` and shifting `temperature.csv` from `dataset_memory/` by a per-recording offset (per-recording confidence in [`dataset_aligned/alignment_offsets.json`](dataset_aligned/alignment_offsets.json#L1)). 10 recordings: 003, 006–014.
2. **Offline labeling** — [`src/pipeline/label.py`](src/pipeline/label.py#L195) drives `src/labeling/run.py` which uses [`src/labeling/align.py`](src/labeling/align.py#L155) to (i) bandpass 20–450 Hz + 50 Hz notch the native 2000 Hz EMG, (ii) square + 50 ms moving-average + sqrt to produce an envelope, (iii) decimate 20:1 to a 100 Hz Unix-time grid via linear interpolation, (iv) attach per-set labels (in_active_set, set_number, exercise from `Participants.xlsx` dual-row schema, RPE, set_phase thirds, rep_count_in_set from joint extrema or marker fallback, rep_density_hz). Output: `data/labeled/recording_NNN/aligned_features.parquet` (26 columns; `t_unix`, biosignals, labels, joint angles).
3. **Window-feature extraction** — [`src/features/window_features.py`](src/features/window_features.py#L152) loads each native CSV again at modality rate (EMG 2000 Hz, Acc 100 Hz, PPG 100 Hz/50 Hz, Temp 1 Hz) and runs [`src/features/emg_features.py`](src/features/emg_features.py), [`src/features/acc_features.py`](src/features/acc_features.py), [`src/features/ppg_features.py`](src/features/ppg_features.py), [`src/features/temp_features.py`](src/features/temp_features.py) with sliding windows (EMG 500 ms / 100 ms; acc 2 s / 100 ms; PPG 10 s / 100 ms; temp 60 s / 100 ms), then `merge_asof`-aligns onto the labeled 100 Hz grid. Per-set aggregates with within-set slope are produced via `_build_set_features`. Adds soft-target columns (`reps_in_window_2s`, `phase_frac_*`). Outputs `<run_dir>/window_features.parquet` and `set_features.parquet`.
4. **Tabular baseline** — [`scripts/train_lgbm.py`](scripts/train_lgbm.py) (despite the name, uses sklearn `RandomForestRegressor`/`RandomForestClassifier` after a refactor) tunes 4 separate models with Optuna + `GroupKFold(n_splits=5)`, groups on **recording_id** (`wf_groups = wf["recording_id"].map(rec_to_int).values`, [`scripts/train_lgbm.py:106`](scripts/train_lgbm.py#L106)), splits read from [`configs/splits_per_fold.csv`](configs/splits_per_fold.csv) (only 9 recordings — recording_003 missing).
5. **NN training** — [`scripts/train_optuna.py`](scripts/train_optuna.py) drives `src/training/loop.py:run_cv` over `WindowFeatureDataset` ([`src/data/datasets.py`](src/data/datasets.py)) or `RawMultimodalWindowDataset` ([`src/data/raw_window_dataset.py`](src/data/raw_window_dataset.py)). Reads CV folds via [`src/training/cv.py:load_or_generate_splits`](src/training/cv.py#L12) from [`configs/splits_loso.csv`](configs/splits_loso.csv) (10 subjects, 10 folds, 1 subject per fold, groups on `subject_id`). 5 architectures × {features, raw} × HP search × 3 phase-2 seeds. Multi-task with optional Kendall uncertainty weighting in [`src/training/losses.py`](src/training/losses.py).
6. **Streaming** — [`src/streaming/realtime.py`](src/streaming/realtime.py) wires per-modality causal extractors ([`src/streaming/emg_streaming.py`](src/streaming/emg_streaming.py), [`acc_streaming.py`](src/streaming/acc_streaming.py), [`ppg_streaming.py`](src/streaming/ppg_streaming.py), [`temp_streaming.py`](src/streaming/temp_streaming.py), `eda_streaming.py`/`ecg_streaming.py` retained for diagnostics) using `CausalBandpass`/`CausalNotch`/`OnlineStats` from [`src/streaming/filters.py`](src/streaming/filters.py) and [`src/streaming/online_stats.py`](src/streaming/online_stats.py).

## Phase 2 — Filtering & alignment audit

### EMG envelope path ([`src/labeling/align.py:155`](src/labeling/align.py#L155))

- **Operation order observed:** `nan-fill → Butterworth 4-th order BP 20–450 Hz (sosfiltfilt) → IIR 50 Hz notch Q=30 (filtfilt) → square → 50 ms moving-average → sqrt → linear interpolation onto 100 Hz grid`. Order is correct (BP+notch run on the raw 2000 Hz signal *before* squaring; sqrt comes *after* the MA) (Merletti & Parker 2004).
- **Anti-aliasing margin for 20:1 decimation:** Nyquist of the 100 Hz target grid is 50 Hz. The 50 ms boxcar has its first null at 20 Hz (sinc function), so power above 20 Hz in the squared signal is attenuated < 1/π² of the DC magnitude before decimation. Marginal but acceptable (Oppenheim & Schafer 2010). A small further improvement: cascade an explicit 4-th order Butterworth LP at 25–30 Hz before the MA, which would harden the anti-alias rolloff.
- **Bandpass before squaring?** Yes — verified at `align.py:193–202`.
- **Filtfilt offline / sosfilt streaming:** offline EMG envelope uses `sosfiltfilt`+`filtfilt` (zero-phase, allowed offline). The streaming counterpart [`src/streaming/emg_streaming.py:107`](src/streaming/emg_streaming.py#L107) uses `CausalBandpass(20.0, 450.0, fs)` + `CausalNotch` (sosfilt + persisted `zi`). Order matches.

### EMG fatigue feature formulas

- MNF formula at [`emg_features.py:189`](src/features/emg_features.py#L189): `mnf = sum(f * pxx) / sum(pxx)` over [20, 450] Hz — matches De Luca 1997.
- MDF formula at lines 192–196: cumulative PSD reaches 50 % of total — matches Phinyomark et al. 2012.
- FInsm5 at lines 200–202: `M(-1) = sum(pxx / max(f,1.0))`, `M(5) = sum(f**5 * pxx)`, ratio = `M(-1) / M(5)` — formula matches Dimitrov et al. 2006 (note: the `np.maximum(f_phys, 1.0)` floor is a numerical-stability hack that biases `M(-1)` downward in the 0–1 Hz bin; bin width ≈ 7.8 Hz at `nperseg=256, fs=2000`, so the lowest bin centred at 0 is the only one affected — minor).
- Welch parameters consistent across MNF/MDF/FInsm5: `nperseg = min(256, len(window))`, default Hann window, no overlap argument (defaults to `nperseg/2`). Welch 1967.
- Window/hop: 500 ms / 100 ms — matches CLAUDE.md.

### Leakage hotspots

| Step | Inside fold? | Notes |
|------|--------------|-------|
| Feature selection (`top_k`, modality ablation) | Outside fold | `scripts/train_with_top_k.py`, `scripts/sweep_top_k.py`, `scripts/ablate_modalities.py` apply selection / ablation **before** CV. For the v17 ablation table, this is OK because the selection criterion (modality identity) is exogenous. For top-K selection it is not — top-K should be re-fit per fold (Saeb et al. 2017, Little et al. 2017). |
| Standardization / scaling (NN path) | **Per-recording, before split** | `WindowFeatureDataset` and `RawMultimodalWindowDataset` compute per-recording (center, scale) at construction time, on each recording's own first 90 s. Because each recording's stats are computed only from that recording's own data, this is leak-safe under subject-wise CV. |
| Per-subject baseline (EMG) | Per-recording, before split | Same as above — uses each recording's own baseline. ✅ |
| Imputation (RF baseline) | **Inside fold** via `sklearn.pipeline.Pipeline([imputer, model])` | Verified at [`scripts/train_lgbm.py:144-149`](scripts/train_lgbm.py#L144). ✅ |
| Optuna HP search | CV-internal in RF (each trial does its own GroupKFold over the same `sf_groups`); CV-internal in NN (`run_one_cv` runs the same outer folds inside each trial). ✅ |

### Multi-task loss weighting

- Strategy: **Kendall uncertainty weighting** (toggled via `--no-uncertainty`; default ON in `train_optuna.py:292`). Implemented as learnable `log_var` parameters per task in `MultiTaskLoss`, with a separate AdamW param-group at `weight_decay=0` for the noise scales ([`src/training/loop.py:257-264`](src/training/loop.py#L257)). Kendall et al. 2018.
- **However**: per `runs/FINAL_REPORT.md`, single-task NNs beat multi-task NNs on every retrained task — suggesting that uncertainty weighting still under-weights some heads in this small-N setting. Single-task fatigue-TCN at 2 s gets r ≈ +0.43 / MAE = 0.79 vs multi-task best (r = +0.36 / MAE = 0.85).
- Justification documented in `runs/FINAL_REPORT.md` ✅ and in the script comments ✅.

### Diagnostics

- **Fatigue:** Per-subject Pearson r is bimodal (Tias +0.84, Hytten −0.62 in [`runs/20260427_110653_default/model_card.md:71-80`](runs/20260427_110653_default/model_card.md#L71)). Hytten's negative correlation flagged for "atypical RPE rating behaviour" but no follow-up. RPE is monotonically increasing within a session (set 1 → set 12), so `set_number` dominates feature importance — that part of the variance is capturable from any time-of-session feature, not from biosignals. The biological residual is what biosignal features must capture, and there the median r ≈ 0.45 is approximately at the literature ceiling for biosignal-only RPE prediction (Borg 1982).
- **Exercise:** Confusion matrix in `plots/cm_exercise.png` (not read in this audit but referenced); per-subject F1 collapses for "lucas" (0.138) — flagged as motion atypicality. Squat ↔ deadlift confusion is expected (similar wrist trajectory) but not quantified in this audit.
- **Phase:** State-machine F1 = 0.306 (well below 0.85 deployment threshold from `motion-rep-detection` skill); ML F1 0.323–0.55 depending on run/single-task. Per CLAUDE.md "State-machine før ML for reps og fase" — the rule is being followed (state-machine evaluated first), and ML is correctly flagged as fallback. The phase classification problem from biosignals alone is genuinely hard (no joint angle in real-time).
- **Reps:** State-machine MAE 2.98 reps; ML 1.26 reps; both above dummy 1.29. Soft-overlap NN fits its own target perfectly but the conversion to integer set count is missing (`scripts/run_singletask_winners.py` may close this gap; not run in this audit).

### State-machine vs ML (phase + reps)

Per the `motion-rep-detection` skill rule, ML is justified only when state-machine F1 < 0.85. Both state-machine F1 (phase 0.31) and ML F1 (phase 0.32–0.55) are below 0.85, which means **neither approach is deployment-grade for phase**. For reps, ML beats state-machine by a wide margin (1.26 vs 2.98 MAE), so the project default of "state-machine first, ML fallback" should flip to ML primary for reps; but RF set-count MAE is barely better than dummy (1.26 vs 1.29) so even ML is not deployment-ready. The cleanest reps story is the v13 single-task LSTM soft-overlap target — the open work is to convert per-window soft-overlap predictions back to integer set count (not done as of this audit).
